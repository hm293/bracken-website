**Brand:** Sacred Sound Sister

## 1. Brand Essence

### Identity

- **Brand Name:** Sacred Sound Sister (singular)
- **Founder:** Bracken Roots
- **Practice:** Sound-based wellbeing - relaxation, self-awareness, and restoration through vibration, voice, and intentional presence
- **Symbol:** The triquetra (Celtic trinity knot) with three S's and fern/botanical details, representing the interconnectedness of mind, body, and spirit

### Tagline Options

- "Rest is the medicine"
- "Sound becomes the guide"
- "Grounding. Restoring. Returning."

### Positioning

Sacred Sound Sister sits between intimate soulfulness and polished credibility - warm, earthy, and deeply personal, but with enough refinement that a corporate wellness manager feels confident reaching out. Not luxury-aspirational. Not esoteric. Grounded and real.

### Target Audiences

1. **Individual clients** - people curious about sound healing, looking for rest and restoration
2. **Corporate wellness** - companies and event organisers seeking wellness experiences for teams
3. **Retreat organisers & collaborators** - other practitioners and venues wanting to book Bracken
4. **Festival & event producers** - looking for immersive sound experiences

The site speaks to all equally. The professionalism of the site itself does the corporate convincing - not corporate language or styling.

## 2. Tone of Voice

### Character

- Warm but not gushing
- Poetic but not vague
- Knowledgeable but not clinical
- Inviting but not salesy
- First person, intimate, grounded

### Language

**Use:** grounding, restoring, intentional, restorative, held, welcoming, weave/woven, sacred, softness, presence, rest, nervous system, restoration, reflection, calm, inclusive

**Avoid:** healing (overused), spiritual journey, chakra, manifest, vibe, energy work, transform your life, unlock, align

### Writing Style

- Short paragraphs with breathing room
- Second person ("you") when addressing the reader
- First person ("I") when Bracken speaks about her practice
- Avoid jargon - make sound healing accessible to newcomers
- Let testimonials do the selling

- **Shadows:** Colour-tinted with low opacity (e.g. `rgba(155, 139, 122, 0.1)`) - never neutral grey shadows.
- **Contrast ratios:** Deep Earth on Ritual Cream = 12.3:1 (passes WCAG AAA). All text combinations verified at minimum AA (4.5:1).

## 4. Typography

Serif-forward, warm, and literary.

### Font Stack

| Role               | Font                    | Weight        | Size Range | Tracking | Line Height |
|--------------------|-------------------------|---------------|------------|----------|-------------|
| Display / Hero     | Cormorant Garamond      | Light (300)   | 48-72px    | -0.03em  | 1.2         |
| Headings           | Cormorant Garamond      | Medium (500)  | 24-36px    | -0.02em  | 1.3         |
| Body               | Lora                    | Regular (400) | 16-18px    | 0        | 1.7         |
| Captions / Meta    | Raleway                 | Regular (400) | 12-14px    | 0.05em   | 1.5         |
| CTA / Navigation   | Raleway                 | Medium (500)  | 14-16px    | 0.08em   | 1.5         |

### Why This Pairing

- **Cormorant Garamond** - Elegant and distinctive without being cold. Has warmth and character that matches Bracken's aesthetic. More refined than Playfair Display.
- **Lora** - Highly readable serif for body text. Gentle organic quality that pairs well with the earthy palette.
- **Raleway** - Clean sans-serif contrast for UI elements without feeling generic. The letter-spacing gives it a refined, editorial quality.

### Rules

- Maximum line length: 65-75 characters for body text
- Minimum body text: 16px (prevents iOS auto-zoom)
- Font loading: `font-display: swap` with Cormorant Garamond preloaded
- All three fonts available on Google Fonts

## 7. The Unforgettable Thing

**The website should feel like the beginning of a session.**

The pace of everything - the load sequence, the scroll rhythm, the transitions - mirrors the cadence of a slow, calm breath. Visitors leave the site feeling slightly more settled than when they arrived.

This is achieved through:
- Slower-than-typical animation durations (500-700ms instead of 200-300ms)
- Breathing rhythm in staggered reveals - elements appear in groups, like inhale-pause-exhale
- Generous negative space between sections - the page rests
- Sound wave motif as section dividers, connecting everything back to sound

## 8. Motion & Animation System

### Design Principle

Every animation is deliberate and unhurried. The motion language mirrors the calming rhythm of a sound bath session. Nothing snaps. Everything settles.

### Motion Tokens

| Token | Value | Usage |
|---|---|---|
| `--duration-reveal` | `600ms` | Section scroll reveals |
| `--duration-hero` | `500ms` | Hero load sequence elements |
| `--duration-stagger` | `300ms` | Gap between hero elements |
| `--duration-stagger-list` | `80ms` | Gap between list/card items |
| `--duration-hover` | `200ms` | Hover/active state transitions |
| `--easing-settle` | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` | Primary easing - a weighted, settling ease-out |
| `--easing-hover` | `ease-out` | Interactive state transitions |

### Page Load Sequence (Hero)

A single orchestrated reveal, staggered like a breath:

```
0ms       Ritual Cream background (instant)
0-500ms   Hero image fades in (opacity 0→1)
300-800ms Gradient overlay settles
600ms     Heading fades up (translateY 20px→0, opacity 0→1)
900ms     Tagline fades up (same transform)
1200ms    CTA fades up (same transform)
```

### Scroll-Triggered Section Reveals

Each section fades up as it enters the viewport using Intersection Observer:

```css
[data-animate] {
  opacity: 0;
  transform: translateY(24px);
}

[data-animate].revealed {
  opacity: 1;
  transform: translateY(0);
  transition: opacity var(--duration-reveal) var(--easing-settle),
              transform var(--duration-reveal) var(--easing-settle);
}

[data-animate-stagger].revealed > *:nth-child(1) { transition-delay: 0ms; }
[data-animate-stagger].revealed > *:nth-child(2) { transition-delay: 80ms; }
[data-animate-stagger].revealed > *:nth-child(3) { transition-delay: 160ms; }
[data-animate-stagger].revealed > *:nth-child(4) { transition-delay: 240ms; }
```

### Reduced Motion

All animations fully disabled when `prefers-reduced-motion: reduce` is active:

```css
@media (prefers-reduced-motion: reduce) {
  [data-animate],
  [data-animate-stagger] > * {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

### Interactive States

Every clickable element must have hover, focus-visible, and active states. No exceptions.

| Element | Hover | Focus-Visible | Active |
|---|---|---|---|
| Nav links | Underline slides in from left (`scaleX 0→1`), 200ms | 2px Burnished Brass outline, offset 4px | Opacity 0.8 |
| CTA buttons | Background warms slightly, subtle glow shadow expands, 200ms | 2px Burnished Brass outline, offset 4px | `scale(0.98)` press feel |
| Offering cards | Lift `translateY(-4px)` + shadow-brand → shadow-brand-lg, 200ms | 2px Burnished Brass outline | `translateY(-2px)` settles back |
| Testimonial nav dots | Colour shifts to Burnished Brass | 2px outline | `scale(0.95)` |
| Footer links | Colour shifts to Burnished Brass | 2px outline | Opacity 0.8 |
| Form inputs | Border colour → Stone | Border → Burnished Brass + outer glow | N/A |
| Submit button | Same as CTA | Same as CTA | Same + loading spinner replaces text |

All transitions use `transform` and `opacity` only. Never `transition-all`. Duration: `200ms ease-out`.

### Sound Wave Section Divider

Sine-wave SVG evokes sound vibration between sections:

```svg
<svg viewBox="0 0 1440 24" preserveAspectRatio="none">
  <path d="M0,12 C120,4 240,20 360,12 C480,4 600,20 720,12 C840,4 960,20 1080,12 C1200,4 1320,20 1440,12"
        stroke="#E8DFD3" fill="none" stroke-width="1"/>
</svg>
```

### Hard Rules

- Only animate `transform` and `opacity`
- Never use `transition-all`
- Never exceed 700ms for any single animation
- All animations must be disabled by `prefers-reduced-motion`
- No autoplay audio or video
- No animation should block user interaction

## 9. Accent Colour Strategy

**Burnished Brass (`#B8976A`) is precious.** It appears in only three places on the entire page:

1. The primary CTA button (hero "Explore" + contact "Get in Touch")
2. Focus-visible outlines (accessibility requirement)
3. The "Let's Work Together" heading in the contact section

Everything else uses the taupe/earth spectrum. This makes the brass moments feel intentional and striking - like the moment a gong is struck in a quiet room.

## 10. Spatial Composition

### Grid-Breaking Moments

Two deliberate asymmetric elements prevent the layout from feeling like a template:

**About section:** The photo extends ~60px beyond its column boundary, overlapping the section edge. Creates visual tension while remaining grounded. On mobile, it bleeds to the screen edge.

**Testimonial quotes:** Oversized decorative quotation marks (Cormorant Garamond, 120px, Soft Linen colour) extend ~20px outside the quote container top-left. Gives the section an editorial quality.

### Depth Layering System

Three defined surface levels - surfaces never sit at the same z-plane:

| Level | Background | Shadow | Z-Index | Usage |
|---|---|---|---|---|
| Base | Ritual Cream `#FAF7F2` | None | `0` | Page background |
| Elevated | Warm Sand `#F0E8DC` | `shadow-brand-sm` | `10` | Cards, testimonial section, offering cards |
| Floating | Ritual Cream `#FAF7F2` | `shadow-brand-lg` | `40-50` | Mobile nav drawer, sticky header on scroll |

### Z-Index Scale

```
--z-base: 0
--z-grain: 1
--z-content: 10
--z-header: 40
--z-nav-drawer: 50
--z-modal: 100
```

## 12. Technical Specifications

### Stack

- **CMS:** Payload CMS 3.x (open source, MIT license)
- **Framework:** Next.js 15 (App Router) - Payload installs directly into the Next.js app
- **Template:** `with-vercel-website` (official Payload starter for Vercel)
- **Database:** Neon Postgres (serverless, free tier - 0.5GB storage)
- **File Storage:** Vercel Blob (with `clientUploads: true` for large image uploads)
- **Hosting:** Vercel (free hobby tier or $20/mo Pro for commercial use)
- **Styling:** TailwindCSS + custom theme tokens matching brand guidelines
- **Fonts:** Google Fonts - Cormorant Garamond, Lora, Raleway
- **Estimated monthly cost:** $0-20

### Content Management (for Bracken)

Bracken can self-manage content via Payload's admin panel:
- Edit all text content through structured fields
- Upload/swap photos with drag-and-drop
- Add, remove, and reorder page sections (block-based layout builder)
- Add new testimonials
- Update offerings and service descriptions
- Live Preview - see changes in real-time before publishing
- Draft/publish workflow - stage changes before going live
- Content changes deploy instantly via on-demand ISR (no rebuild needed)

Developer-only changes (requires code update):
- Brand colours, fonts, or overall design system
- Adding entirely new section/block types
- Modifying site structure or navigation patterns

### Payload Collections (Data Model)

| Collection | Purpose | Editable Fields |
|---|---|---|
| **Pages** | Main site pages (Home, About, Offerings) | Title, slug, layout blocks, SEO meta |
| **Media** | All images and assets | Upload, alt text, focal point crop |
| **Testimonials** | Client quotes | Quote text, name, context/event |
| **Offerings** | Services and experiences | Title, description, duration, category |
| **Events** | Past/upcoming events | Name, venue, date, photo |

### Layout Blocks (Page Builder)

Custom blocks Bracken can add/reorder on any page:

| Block | Description |
|---|---|
| **Hero** | Full-viewport image with text overlay |
| **About** | Bio section with photo and text |
| **Offerings Grid** | Card layout of services by category |
| **Testimonial Carousel** | Rotating client quotes |
| **Gallery** | Photo grid with optional venue logos |
| **Contact** | Contact form + direct contact details |
| **Content** | Generic rich text + media block |
| **CTA** | Call-to-action banner |

### Performance

- Next.js ISR (Incremental Static Regeneration) for instant content updates
- Vercel CDN edge network for global delivery
- Vercel Blob CDN for image delivery
- Lazy load all below-fold images (`loading="lazy"`)
- Preload hero image and Cormorant Garamond font
- `font-display: swap` to prevent invisible text
- Next.js Image component for automatic WebP/AVIF optimisation and responsive srcsets
- Target: Largest Contentful Paint < 2.5s

### Accessibility

- All contrast ratios minimum WCAG AA (4.5:1)
- `prefers-reduced-motion` respected - parallax and scroll animations disabled
- Alt text on all images
- Keyboard navigable (visible focus states in Burnished Brass)
- Touch targets minimum 44px
- Skip-to-content link
- Semantic heading hierarchy (h1 > h2 > h3)
